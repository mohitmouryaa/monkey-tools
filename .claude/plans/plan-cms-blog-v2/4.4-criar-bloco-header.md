# ✅ 4.4 — Criar bloco renderer: Header

## Contexto
Renderiza um bloco do tipo `header` salvo pelo `@editorjs/header`.
Server component (P2 = A). Mapeia `level` (1–6) para a tag HTML
correspondente (`<h1>`–`<h6>`).

## Detalhe técnico
Criar `apps/web/modules/blog/ui/components/blocks/header-block.tsx`.

Shape do bloco salvo pelo Editor.js:
```json
{ "type": "header", "data": { "text": "Título", "level": 2 } }
```

```tsx
interface HeaderBlockData {
  text?: string;
  level?: number; // 1..6
}

export const HeaderBlock = ({ data }: { data: HeaderBlockData }) => {
  const level = Math.min(Math.max(data.level ?? 2, 1), 6);
  const Tag = `h${level}` as "h1" | "h2" | "h3" | "h4" | "h5" | "h6";
  const text = data.text ?? "";
  return (
    <Tag
      // biome-ignore lint/security/noDangerouslySetInnerHtml: Editor.js inline tools (bold, italic, link) já produzem HTML sanitizado pelo próprio EditorJS no admin
      dangerouslySetInnerHTML={{ __html: text }}
    />
  );
};
```

Notas:
- Editor.js permite inline tools (bold, italic, link) — o `text` já vem
  com tags inline. Por isso `dangerouslySetInnerHTML`. Sanitização é
  responsabilidade do Editor.js no admin.
- Não aplicar classes Tailwind aqui — o wrapper externo (4.13) usa
  `prose` que estiliza headings automaticamente.
