# ✅ 5.1 — Adicionar `featuredPostId` no `createToolSchema`

## Contexto

O schema Zod consumido pelo `tool-form.tsx` e pelas mutations `tools.create`/`tools.update` ainda não inclui `featuredPostId`. Sem esse campo, 5.3/5.4/5.5 não funcionam.

## Detalhe técnico

Editar `apps/web/modules/dashboard/schema/tool.tsx`. No objeto passado para `z.object({...})` de `createToolSchema`, adicionar (após `isActive`, mantendo ordem aproximada do form):

```ts
featuredPostId: z.string().nullable().optional(),
```

Decisão **P8=B**: aceita string id válida, `null` explícito ou ausência (`undefined`). O combobox da 5.2 vai passar `""` quando o usuário escolher "Sem post em destaque" — a tradução `"" → null` acontece nas mutations (5.4 e 5.5), não no schema.

Não exportar tipo separado — o tipo inferido (`z.infer<typeof createToolSchema>`) já cobre.

## Arquivos tocados

- `apps/web/modules/dashboard/schema/tool.tsx` — adicionar campo `featuredPostId`.
