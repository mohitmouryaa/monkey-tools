# ✅ 5.4 — Gravar `featuredPostId` no `tools.create`

## Contexto

A procedure `tools.create` em `toolsRouter.ts` lista campos explícitos ao construir o `new ToolModel({...})`. Como `featuredPostId` é novo, precisa ser explicitado lá para que o save persista corretamente.

## Detalhe técnico

Editar `apps/web/trpc/routers/toolsRouter.ts`. No objeto passado para `new ToolModel({...})` dentro de `create`, adicionar (mantendo o estilo do arquivo):

```ts
featuredPostId: input.featuredPostId && input.featuredPostId !== "" ? input.featuredPostId : null,
```

Decisão **P3=A** + **P8=B**: trata `""`, `null`, `undefined` como ausência → grava `null`.

## Arquivos tocados

- `apps/web/trpc/routers/toolsRouter.ts` — adicionar linha em `create`.

## Verificação

- [ ] `pnpm --filter web typecheck` passa.
- [ ] Criar uma tool nova com `featuredPostId` preenchido grava o id na coleção (inspecionar via `tools.getOne` ou no Mongo).
- [ ] Criar uma tool nova sem `featuredPostId` grava `null` (não `undefined`).

## Decisões aplicadas

- **P3=A** + **P8=B**: traduz `""`/`undefined`/`null` para `null` antes do save.
