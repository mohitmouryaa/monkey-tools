# ✅ 4.10 — Criar bloco renderer: Checklist

## Contexto
Renderiza um bloco `checklist` salvo pelo `@editorjs/checklist`. No
público o checklist é read-only (checkboxes desabilitados), conforme
definido no plano. Server component (P2 = A).

## Detalhe técnico
Criar `apps/web/modules/blog/ui/components/blocks/checklist-block.tsx`.

Shape:
```json
{
  "type": "checklist",
  "data": {
    "items": [
      { "text": "Item feito", "checked": true },
      { "text": "Item pendente", "checked": false }
    ]
  }
}
```

```tsx
interface ChecklistItem {
  text?: string;
  checked?: boolean;
}

interface ChecklistBlockData {
  items?: ChecklistItem[];
}

export const ChecklistBlock = ({ data }: { data: ChecklistBlockData }) => {
  const items = data.items ?? [];
  if (items.length === 0) return null;

  return (
    <ul className="not-prose my-6 space-y-2 list-none pl-0">
      {items.map((item, idx) => (
        <li key={idx} className="flex items-start gap-2">
          <input
            type="checkbox"
            checked={!!item.checked}
            disabled
            className="mt-1 size-4 rounded border-border accent-primary"
            aria-label={item.text}
            readOnly
          />
          <span
            className="text-foreground"
            // biome-ignore lint/security/noDangerouslySetInnerHtml: Editor.js inline tools sanitiza no admin
            dangerouslySetInnerHTML={{ __html: item.text ?? "" }}
          />
        </li>
      ))}
    </ul>
  );
};
```

Notas:
- `not-prose` neutraliza estilos de lista do `prose` (que renderiza
  bullets — não queremos aqui pois o checkbox substitui o marcador).
- `disabled` + `readOnly` evita interação client-side (renderer é RSC).
