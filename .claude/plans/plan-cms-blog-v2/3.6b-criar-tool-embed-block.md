# ✅ 3.6b — Criar plugin custom Editor.js: Tool Embed

## Contexto
Plugin que persiste `{ toolId: string }` no JSON e renderiza um seletor de tool dentro do bloco. Decisão P7: usar `queryClient.fetchQuery` para acessar o cache TanStack já populado.

## Detalhe técnico
Criar `apps/web/modules/dashboard/ui/components/editor-js/tool-embed-block.tsx`:

```ts
import type { QueryClient } from "@tanstack/react-query";

interface ToolEmbedData {
  toolId: string;
}

interface ToolEmbedConfig {
  queryClient: QueryClient;
}

interface ToolItem {
  _id: string;
  title: string;
}

interface ToolsListResponse {
  items: ToolItem[];
}

export class ToolEmbedBlock {
  private data: ToolEmbedData;
  private config: ToolEmbedConfig;
  private wrapper: HTMLDivElement | null = null;

  constructor({ data, config }: { data: ToolEmbedData; config: ToolEmbedConfig }) {
    this.data = data ?? { toolId: "" };
    this.config = config;
  }

  static get toolbox() {
    return {
      title: "Tool Embed",
      icon: "<svg width='17' height='15'><path d='M2 5h13M2 10h13' stroke='currentColor' stroke-width='2'/></svg>",
    };
  }

  render(): HTMLElement {
    const wrapper = document.createElement("div");
    wrapper.className = "p-3 border rounded-md bg-muted/30";
    this.wrapper = wrapper;

    const select = document.createElement("select");
    select.className = "w-full p-2 border rounded bg-background text-foreground";
    select.disabled = true;
    select.innerHTML = '<option value="">Loading tools...</option>';

    select.addEventListener("change", () => {
      this.data.toolId = select.value;
    });

    wrapper.appendChild(select);

    // Fetch tools via TanStack Query (cache compartilhado com o app)
    this.loadTools(select).catch((err) => {
      console.error("ToolEmbedBlock: failed to load tools", err);
      select.innerHTML = '<option value="">Failed to load tools</option>';
    });

    return wrapper;
  }

  private async loadTools(select: HTMLSelectElement): Promise<void> {
    const queryKey = ["tools", "getMany", { input: { pageSize: 100 } }];
    // Em vez de reconstruir queryKey manualmente, usar trpc helper se acessível.
    // Como o plugin não tem acesso direto a `trpc`, usar fetchQuery com queryFn explícita
    // via fetch ao endpoint tRPC. Alternativa pragmática: passar uma função `fetchTools`
    // já bound via config (mais simples, evita dependência de queryKey internals).
    //
    // ATENÇÃO: Se precisar acoplar ao queryKey gerado pelo tRPC, ajuste o wrapper (3.6a)
    // para passar `config.fetchTools` em vez de `config.queryClient` direto.
    const data = await this.config.queryClient.fetchQuery<ToolsListResponse>({
      queryKey,
    });

    select.disabled = false;
    select.innerHTML = '<option value="">— Select a tool —</option>';
    for (const tool of data.items) {
      const opt = document.createElement("option");
      opt.value = tool._id;
      opt.textContent = tool.title;
      if (tool._id === this.data.toolId) opt.selected = true;
      select.appendChild(opt);
    }
  }

  save(blockContent: HTMLElement): ToolEmbedData {
    const select = blockContent.querySelector("select");
    return { toolId: select?.value ?? this.data.toolId };
  }

  validate(savedData: ToolEmbedData): boolean {
    return Boolean(savedData?.toolId);
  }
}
```

**Atenção crítica sobre `queryKey` com tRPC**: o tRPC v11 + `@trpc/tanstack-react-query` gera `queryKey` numa estrutura específica. Reconstruir manualmente é frágil. Se a implementação acima não acertar a queryKey, **a estratégia robusta é**:

1. No wrapper (3.6a), em vez de `config: { queryClient }`, passar `config: { fetchTools }`, onde `fetchTools = () => queryClient.fetchQuery(trpc.tools.getMany.queryOptions({ pageSize: 100 }))`.
2. No bloco, chamar `await this.config.fetchTools()`.

Esse refactor é simples e evita acoplar o plugin a internals do tRPC. **Recomendado fazer já assim** — atualize 3.6a e 3.6b para esse contrato:

```ts
// em 3.6a:
toolEmbed: { class: ToolEmbedBlock, config: { fetchTools: () => queryClient.fetchQuery(trpc.tools.getMany.queryOptions({ pageSize: 100 })) } }
// em 3.6b:
interface ToolEmbedConfig { fetchTools: () => Promise<ToolsListResponse> }
```

Adote o segundo formato no commit final.
