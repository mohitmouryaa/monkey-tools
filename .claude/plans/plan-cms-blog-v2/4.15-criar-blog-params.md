# ✅ 4.15 — Criar `blog-params.tsx` (loader nuqs)

## Contexto
Definição central dos query params do `/blog`: `page`, `q`, `tool`.
Espelha o padrão de `modules/dashboard/posts/posts-params.tsx` mas com
nomes adaptados ao público (`q` em vez de `search`).

## Detalhe técnico
Criar `apps/web/modules/blog/blog-params.tsx`.

```tsx
import { parseAsInteger, parseAsString } from "nuqs/server";
import { PAGINATION } from "@/modules/common/constants";

export const blogParams = {
  page: parseAsInteger.withDefault(PAGINATION.DEFAULT_PAGE).withOptions({ clearOnDefault: true }),
  pageSize: parseAsInteger.withDefault(PAGINATION.DEFAULT_PAGE_SIZE).withOptions({ clearOnDefault: true }),
  q: parseAsString.withDefault("").withOptions({ clearOnDefault: true }),
  tool: parseAsString.withOptions({ clearOnDefault: true }),
};
```

Notas:
- Não há `status` (não é exposto no público — listagem só retorna
  `published`).
- `tool` (ID da tool) é string opcional sem default — quando não
  presente, `posts.list` ignora o filtro.
- `pageSize` mantido configurável via URL (mas raramente alterado por
  usuário) — o `blog/page.tsx` usa esse valor para popular `caller.posts.list`.
