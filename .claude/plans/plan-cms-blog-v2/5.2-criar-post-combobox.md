# ✅ 5.2 — Criar `<PostCombobox />`

## Contexto

O `tool-form.tsx` precisa de um combobox de seleção única de post publicado para o campo `featuredPostId`. Não existe combobox genérico de post no projeto. Criamos aqui um componente focado e reusável.

## Detalhe técnico

Criar `apps/web/modules/dashboard/ui/components/post-combobox.tsx`. Marcar como `"use client"`.

**Props:**

```ts
interface PostComboboxProps {
  value: string | null | undefined;
  onChange: (value: string | null) => void;
  disabled?: boolean;
}
```

**Comportamento:**

- Carregar lista única via TanStack Query + tRPC: `useQuery(trpc.posts.list.queryOptions({ status: PostStatus.PUBLISHED, pageSize: 50, page: 1, search: "" }))`.
- Item sentinela no topo: `{ _id: "", title: "Sem post em destaque" }` (P3=A).
- Renderizar usando o `Select` do shadcn (`@workspace/ui/components/select`) — já em uso em `tool-form.tsx`. Para 50 itens, `Select` simples basta.
- `onChange` propaga o `_id` selecionado, **mas mapeia `""` para `null`** antes de chamar `onChange`.
- `value === null | undefined` mostra o trigger com placeholder "Sem post em destaque".
- Se `disabled` ou `isLoading`, desabilitar o trigger.

> **Nota.** Se quiser explorar `Command` (combobox com filtro digitando) no futuro, hoje o stack do admin não tem `Command` instalado. Manter `Select` agora; abrir issue se a lista crescer.

## Arquivos tocados
