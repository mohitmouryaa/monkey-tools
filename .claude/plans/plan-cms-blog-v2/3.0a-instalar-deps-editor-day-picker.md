# ✅ 3.0a — Instalar dependências (Editor.js + react-day-picker)

## Contexto
Fase 3 introduz Editor.js (rich content de posts) e DatePicker shadcn (campo `publishedAt`). Ambos exigem libs que ainda não estão no monorepo.

## Detalhe técnico
Em `apps/web/package.json`, adicionar a `dependencies` (manter ordem alfabética dentro do bloco quando possível):

```
"@editorjs/checklist": "^1.6.0",
"@editorjs/editorjs": "^2.31.0",
"@editorjs/embed": "^2.7.6",
"@editorjs/header": "^2.8.8",
"@editorjs/image": "^2.10.2",
"@editorjs/list": "^2.0.6",
"@editorjs/quote": "^2.7.2",
"@editorjs/table": "^2.4.4",
"react-day-picker": "^9.8.0"
```

(Versões exatas: usar `latest` resolvido via `pnpm add` rodado a partir de `apps/web`.)

Em `packages/ui/package.json` (já há `cmdk` — não tocar ali; `react-day-picker` fica em `apps/web` porque o `<Calendar>` shadcn é um componente — pode ficar em `packages/ui`. Se a convenção do projeto é deixar deps de UI em `packages/ui`, adicionar `react-day-picker` em `packages/ui/package.json` e remover de `apps/web/package.json`. **Conferir convenção rodando `grep -l '"date-fns"' packages/ apps/` antes — onde estiver `date-fns` é a referência.**)

Comandos:
1. `pnpm --filter web add @editorjs/editorjs @editorjs/header @editorjs/list @editorjs/quote @editorjs/embed @editorjs/table @editorjs/checklist @editorjs/image`
2. Decidir local de `react-day-picker` conforme nota acima e rodar `pnpm --filter <pkg> add react-day-picker`.
3. `pnpm install` na raiz para reconciliar lockfile.

## Arquivos tocados
- `apps/web/package.json` — adicionar deps `@editorjs/*` e (possivelmente) `react-day-picker`.
- `packages/ui/package.json` — adicionar `react-day-picker` se a convenção mandar.
- `pnpm-lock.yaml` — atualizado pelo `pnpm install`.

## Verificação
