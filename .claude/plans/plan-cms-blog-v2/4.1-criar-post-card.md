# ✅ 4.1 — Criar `post-card.tsx`

## Contexto
Componente de cartão de post para listagem do `/blog` e da seção de
featured (`featured-posts-section.tsx`). Mostra cover, title, excerpt
e badges das tools associadas. É server component (RSC) por consistência
com o resto da Fase 4 (P2 = A).

## Detalhe técnico
Criar componente em
`apps/web/modules/blog/ui/components/post-card.tsx`.

Props:
```ts
interface PostCardProps {
  post: {
    _id: string;
    slug: string;
    title: string;
    excerpt: string;
    coverImage: string;
    publishedAt?: string | Date | null;
    tools?: Array<{ _id: string; title: string }>;
  };
}
```

- Server component (sem `"use client"`).
- Wrap em `<Link href={\`/blog/${post.slug}\`} className="block group">`.
- Card visual:
  - cover via `<Image src={post.coverImage} alt={post.title} fill sizes="(min-width: 1024px) 33vw, 100vw" className="object-cover" />` num wrapper `relative aspect-[16/9] overflow-hidden rounded-xl`.
  - título `<h3 className="mt-4 text-lg font-semibold text-foreground group-hover:text-primary transition-colors line-clamp-2">`.
  - excerpt `<p className="mt-2 text-sm text-muted-foreground line-clamp-3">`.
  - badges das tools: render apenas até 3 primeiras com `<span className="text-xs px-2 py-1 rounded-full bg-muted text-muted-foreground">{tool.title}</span>`. Esconder badges se `tools.length === 0`.
- Estilo geral do card alinhado a `tool-card.tsx` (mesmo `bg-card border border-border rounded-2xl p-4 transition hover:border-primary` quando aplicável).

Importante: NÃO formatar a `publishedAt` neste card (a Fase 4 não exige
data visível). Se quiser mostrar "Publicado em ...", deixar para fase
posterior — não inventar formato agora.
