# ✅ 5.7 — Renderizar seção "Aprenda mais" na página pública da Tool

## Contexto

Com o cache em vigor (5.6), inserimos a seção que consome o `featuredPostId` da Tool e renderiza um `<PostCard />` linkando para `/blog/[slug]`.

## Detalhe técnico

Editar `apps/web/modules/tools/ui/views/tool-view.tsx`:

1. Importar:
   ```ts
   import { PostCard } from "@/modules/blog/ui/components/post-card";
   ```

2. **Após** o `category` ser resolvido e a `toolData` ser encontrada (`category.tools.find(...)`), buscar o post (se houver):

   ```ts
   let featuredPost: Awaited<ReturnType<typeof caller.posts.getById>> | null = null;
   if (toolData.featuredPostId) {
     try {
       featuredPost = await caller.posts.getById({ id: String(toolData.featuredPostId) });
     } catch {
       featuredPost = null;
     }
   }
   ```

   (Decisão **P7=A**: try/catch para não quebrar visitante anônimo se a procedure protected falhar.)

3. **Posicionamento**: dentro do JSX retornado, **entre o bloco `{toolData.richContent && ...}` e o bloco `{toolData.faqs && ...}`** (na coluna `flex-1`):

   ```tsx
   {featuredPost && (
     <div className="mb-8">
       <h2 className="mb-6 text-2xl font-bold">Aprenda mais</h2>
       <div className="grid grid-cols-1 gap-6 md:max-w-md">
         <PostCard
           post={{
             _id: featuredPost._id,
             slug: featuredPost.slug,
             title: featuredPost.title,
             excerpt: featuredPost.excerpt,
             coverImage: featuredPost.coverImage,
             publishedAt: featuredPost.publishedAt,
             tools: featuredPost.tools as Array<{ _id: string; title: string }>,
           }}
         />
       </div>
     </div>
   )}
   ```

   (Decisão **P6=A**: posição entre `richContent` e `FAQs`. Card em coluna estreita `md:max-w-md`.)

   > **Ajuste fino do shape do `PostCard`.** Se o `PostCard` já criado na Fase 4 receber outro shape de `post` (ex: já espera o tipo direto retornado por `posts.list`), simplificar passando o objeto retornado por `getById` direto: `<PostCard post={featuredPost} />`. Verificar a assinatura real antes de aplicar a versão verbose acima.

## Arquivos tocados

## Verificação

- [ ] `pnpm --filter web typecheck` passa.
- [ ] **Logado**: na página pública de uma tool com `featuredPostId` setado, a seção "Aprenda mais" aparece entre rich content e FAQs com um `PostCard` linkado para `/blog/[slug]`.
- [ ] **Anônimo (browser private/incógnito)**: abrir a mesma URL e confirmar que a seção aparece.
- [ ] Tool sem `featuredPostId`: seção não renderiza (nem header, nem card).
- [ ] Tool com `featuredPostId` apontando para post deletado: seção não renderiza (try/catch + fallback `null`).

## Caso degradado (P7)

Se no teste anônimo a seção **não** aparecer (porque `posts.getById` é `protectedProcedure` e exige sessão):

- **NÃO marcar `✅`** nesta task.
- **PARAR** a fase imediatamente.
- Reportar ao usuário com a mensagem:
  > "P7 caiu no caso degradado: visitantes anônimos não veem a seção 'Aprenda mais' porque `posts.getById` requer sessão. Necessário criar procedure pública dedicada (`posts.getByIdPublic` filtrando PUBLISHED + publishedAt<=now) ou popular o post diretamente em `getCategoryWithTools`. Out-of-scope desta fase — abrir nova fase."

## Decisões aplicadas

- **P6=A**: posição entre `richContent` e `FAQs`.
- **P7=A**: `caller.posts.getById` server-side, com try/catch e teste explícito anônimo.
