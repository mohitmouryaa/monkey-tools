# ✅ 4.11 — Criar bloco renderer: Image

## Contexto
Renderiza um bloco `image` salvo pelo `@editorjs/image` (que usa nosso
endpoint `/api/upload?prefix=cms`). Usar `next/image` (consistente com
o resto do projeto). Server component (P2 = A).

## Detalhe técnico
Criar `apps/web/modules/blog/ui/components/blocks/image-block.tsx`.

Shape:
```json
{
  "type": "image",
  "data": {
    "file": { "url": "https://.../cms/abc.png" },
    "caption": "Legenda",
    "withBorder": false,
    "withBackground": false,
    "stretched": false
  }
}
```

```tsx
import Image from "next/image";

interface ImageBlockData {
  file?: { url?: string };
  caption?: string;
  withBorder?: boolean;
  withBackground?: boolean;
  stretched?: boolean;
}

export const ImageBlock = ({ data }: { data: ImageBlockData }) => {
  const url = data.file?.url;
  if (!url) return null;

  return (
    <figure className="my-6">
      <div
        className={[
          "relative w-full overflow-hidden rounded-xl",
          data.withBorder ? "border border-border" : "",
          data.withBackground ? "bg-muted p-4" : "",
        ]
          .filter(Boolean)
          .join(" ")}
        // proporção mínima (4:3) para evitar layout shift quando next/image fill é usado
        style={{ aspectRatio: "16/9" }}
      >
        <Image
          src={url}
          alt={data.caption ?? ""}
          fill
          sizes="(min-width: 1024px) 800px, 100vw"
          className="object-contain"
        />
      </div>
      {data.caption && (
        <figcaption
          className="mt-2 text-sm text-center text-muted-foreground"
          // biome-ignore lint/security/noDangerouslySetInnerHtml: caption do Editor.js
          dangerouslySetInnerHTML={{ __html: data.caption }}
        />
      )}
    </figure>
  );
};
```

Notas:
- `object-contain` em vez de `cover` para não cortar a imagem original.
- `aspect-ratio: 16/9` é o default; `stretched: true` poderia mudar mas
  no v1 mantemos uniforme.
- `next/image` requer que o domínio do bucket S3 esteja em
  `next.config.js > images.remotePatterns`. Se o domínio do
  `DO_SPACES_ENDPOINT` ainda não estiver lá, **NÃO** alterar
  `next.config.js` nesta task — registrar como follow-up. (A Premissa 2
  do plano garante URL pública navegável; se aparecer erro de domínio em
  dev, parar e perguntar.)
