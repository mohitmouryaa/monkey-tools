# ✅ 5.6 — Envolver fetch da Tool em `unstable_cache` com tag `tool:${link}`

## Contexto

Para que a Fase 6.3 consiga revalidar a página pública da Tool ao mudar `featuredPostId`, a chamada atual `caller.categories.getCategoryWithTools(...)` em `tool-view.tsx` precisa estar dentro de um `unstable_cache` com tag por tool.

Hoje a página da tool **não** está envolvida em `unstable_cache` (premissa 10 do plano). Esta task é o pré-requisito da Fase 6.3.

## Detalhe técnico

### Passo 1 — criar helper compartilhado

Criar `apps/web/modules/tools/lib/cache.ts`:

```ts
/**
 * Cria a tag de cache para uma tool baseada em seu `link`.
 * O `link` pode vir com ou sem barra inicial; normalizamos removendo
 * a barra para que a tag seja determinística.
 */
export const tagForTool = (link: string) => `tool:${link.replace(/^\//, "")}`;
```

(Decisão **P9=B** — compartilhado com Fase 6.3.)

### Passo 2 — envolver fetch em `tool-view.tsx`

Editar `apps/web/modules/tools/ui/views/tool-view.tsx`.

1. Adicionar imports:
   ```ts
   import { unstable_cache } from "next/cache";
   import { tagForTool } from "@/modules/tools/lib/cache";
   ```

2. **Manter** a busca atual em duas etapas (fetch da `category` → `find` da tool dentro de `category.tools`), mas mover a chamada de `caller.categories.getCategoryWithTools` para dentro de `unstable_cache`. A tag deriva do segmento da rota (`tool` vem de `params`), que bate exatamente com o `link` normalizado da tool — assim quando 6.3 invocar `revalidateTag(tagForTool(updatedTool.link))`, a tag casará.

   Substituir o fetch atual por:

   ```ts
   const category = await unstable_cache(
     async () => caller.categories.getCategoryWithTools({ slug: toolCategory }),
     ["tool-category-with-tools-v1", toolCategory, tool],
     { revalidate: 3600, tags: [tagForTool(tool)] },
   )();
   ```

   - **keyParts** (P2=A no contexto da explosão original): `["tool-category-with-tools-v1", toolCategory, tool]` — params da URL.
   - **revalidate** (P5=A): `3600` (1h) como TTL fallback.
   - **tags** (P4=A): `[tagForTool(tool)]` onde `tool` é o segmento da URL (= link normalizado).

## Arquivos tocados

- `apps/web/modules/tools/lib/cache.ts` — novo arquivo (helper `tagForTool`).
- `apps/web/modules/tools/ui/views/tool-view.tsx` — imports + cache wrapper.

- [ ] `pnpm --filter web typecheck` passa.

- [ ] Em dev, hits subsequentes ao mesmo `(toolCategory, tool)` não chamam o banco (logar dentro do callback do `unstable_cache`, ou observar latência cair entre o 1º e 2º request).
- [ ] Sanity check do helper:
  - `tagForTool("/pdf-to-jpg") === "tool:pdf-to-jpg"`
  - `tagForTool("pdf-to-jpg") === "tool:pdf-to-jpg"`

## Decisões aplicadas

- **P4=A**: tag `tool:${link.replace(/^\//, "")}`.
- **P5=A**: `revalidate: 3600`.
- **P9=B**: helper em `apps/web/modules/tools/lib/cache.ts`, compartilhado com 6.3.
