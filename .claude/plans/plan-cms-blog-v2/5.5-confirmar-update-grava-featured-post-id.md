# ✅ 5.5 — Confirmar que `tools.update` grava `featuredPostId`

## Contexto

A procedure `tools.update` faz `findByIdAndUpdate(id, { $set: finalUpdateData })`, onde `finalUpdateData` é construído a partir de `input.data` com remap de `categoryId → category`. Como o schema agora inclui `featuredPostId` (5.1), o spread de `input.data` já carrega o campo automaticamente — mas precisamos normalizar `""` → `null` antes do save.

## Detalhe técnico

Editar `apps/web/trpc/routers/toolsRouter.ts`. Em `update`, **ANTES** do `findByIdAndUpdate`, normalizar `featuredPostId`:

```ts
if ("featuredPostId" in finalUpdateData) {
  const v = finalUpdateData.featuredPostId;
  finalUpdateData.featuredPostId = v && v !== "" ? v : null;
}
```

Se o TypeScript reclamar do tipo do helper local (algo como `Omit<typeof updateData, "categoryId"> & { category?: string }`), adicionar `featuredPostId?: string | null` na intersect.

## Arquivos tocados

- `apps/web/trpc/routers/toolsRouter.ts` — adicionar normalização em `update`.

## Verificação

- [ ] `pnpm --filter web typecheck` passa.
- [ ] Editar uma tool no admin alterando o post em destaque persiste o novo id.
- [ ] Editar uma tool selecionando "Sem post em destaque" grava `null`.
- [ ] No banco, `db.tools.findOne({ _id: ... }).featuredPostId` reflete a última seleção (id válido ou `null`, nunca `""` nem `undefined`).
