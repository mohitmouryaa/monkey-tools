# ✅ 5.3 — Adicionar campo "Featured Post" no `tool-form.tsx`

## Contexto

Com o schema atualizado (5.1) e o combobox criado (5.2), o form da Tool no admin ganha o campo de post em destaque.

## Detalhe técnico

Editar `apps/web/modules/dashboard/ui/components/tool-form.tsx`:

1. Importar `PostCombobox`:
   ```ts
   import { PostCombobox } from "./post-combobox";
   ```

2. Adicionar `featuredPostId: null` (ou omitir, deixando `undefined`) nos `defaultValues` iniciais E no objeto passado para `form.reset(...)` no `useEffect` que popula a partir da tool existente.

3. Inserir nova subseção **logo após o fechamento do `<div>` de "General Information" e ANTES do bloco "Visual Configuration"** (P2=A):

   ```tsx
   <div className="space-y-4">
     <h3 className="text-lg font-medium text-foreground">Featured Post</h3>
     <FormField
       control={form.control}
       name="featuredPostId"
       render={({ field }) => (
         <FormItem>
           <FormLabel>Post em destaque</FormLabel>
           <FormDescription>
             Este post aparecerá na seção "Aprenda mais" da página pública da tool.
           </FormDescription>
           <FormControl>
             <PostCombobox
               value={field.value ?? null}
               onChange={field.onChange}
               disabled={disabled}
             />
           </FormControl>
           <FormMessage />
         </FormItem>
       )}
     />
   </div>
   ```

## Arquivos tocados

- `apps/web/modules/dashboard/ui/components/tool-form.tsx` — import + defaultValues + reset + nova subseção.

## Verificação

- [ ] `pnpm --filter web typecheck` passa.
