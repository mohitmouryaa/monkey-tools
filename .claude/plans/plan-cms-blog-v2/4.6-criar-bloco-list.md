# ✅ 4.6 — Criar bloco renderer: List

## Contexto
Renderiza um bloco `list` salvo pelo `@editorjs/list` (v2). Server
component (P2 = A).

## Detalhe técnico
Criar `apps/web/modules/blog/ui/components/blocks/list-block.tsx`.

Shape do bloco (v2 do plugin):
```json
{
  "type": "list",
  "data": {
    "style": "ordered" | "unordered" | "checklist",
    "items": [
      { "content": "texto", "items": [/* recursivo */], "meta": { "checked": true } }
    ]
  }
}
```

⚠️ Atenção: o plugin `@editorjs/list` v2 (instalado no projeto) já suporta
nested lists com `items[].items` recursivos. Renderizar recursivamente.

```tsx
interface ListItem {
  content?: string;
  items?: ListItem[];
  meta?: { checked?: boolean };
}

interface ListBlockData {
  style?: "ordered" | "unordered" | "checklist";
  items?: ListItem[];
}

const renderItems = (items: ListItem[], style: ListBlockData["style"]): React.ReactNode => {
  return items.map((item, idx) => (
    <li key={idx}>
      {/* biome-ignore lint/security/noDangerouslySetInnerHtml: Editor.js inline tools sanitiza no admin */}
      <span dangerouslySetInnerHTML={{ __html: item.content ?? "" }} />
      {item.items && item.items.length > 0 && (
        style === "ordered"
          ? <ol>{renderItems(item.items, style)}</ol>
          : <ul>{renderItems(item.items, style)}</ul>
      )}
    </li>
  ));
};

export const ListBlock = ({ data }: { data: ListBlockData }) => {
  const items = data.items ?? [];
  if (items.length === 0) return null;

  if (data.style === "ordered") {
    return <ol>{renderItems(items, "ordered")}</ol>;
  }
  // unordered ou checklist (delega visual ao Tailwind prose; checklist
  // não tem checkbox aqui — caso venha como style="checklist", trata
  // como unordered. O bloco "checklist" oficial é tratado em 4.10).
  return <ul>{renderItems(items, data.style)}</ul>;
};
```

Notas:
- Não confundir com o block `checklist` separado do `@editorjs/checklist`
  (4.10). O `style: "checklist"` aqui é uma variante do plugin `list` v2
  que ainda não é prioritária — aceitamos sem checkbox no v1.
