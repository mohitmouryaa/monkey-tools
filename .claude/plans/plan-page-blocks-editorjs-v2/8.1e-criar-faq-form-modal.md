✅ # 8.1e — Criar `FaqFormModal`

## Contexto

Form RHF + Zod para `FaqBlockData`. Estrutura quase igual a 8.1c/8.1d — array dinâmico via `useFieldArray` — mas item é `{ question, answer }`. Campo `answer` é HTML legado simples (espelha o renderer 5.4 que faz `dangerouslySetInnerHTML` com classes `[&_a]:text-primary [&_a]:underline [&_strong]:font-semibold`). Por simplicidade, o input é um `<Textarea>` — operador escreve HTML cru se quiser markup ou texto puro caso queira só parágrafo.

## Detalhe técnico

Criar `apps/web/modules/dashboard/ui/components/editor-js-field/faq-form-modal.tsx` (`"use client"`).

Mesmo skeleton de 8.1c, trocando:

- `name="steps"` → `name="items"`
- Schema: `faqBlockDataSchema`
- Tipo: `FaqBlockData`
- `defaultValues`: `{ items: [], ...initialData }`
- Sem `subtitle`. Manter campo `title` (opcional).
- `addItem` default: `{ question: "", answer: "" }`

Por item:

```tsx
<FormField
  control={form.control}
  name={`items.${index}.question`}
  render={({ field }) => (
    <FormItem>
      <FormLabel>Pergunta</FormLabel>
      <FormControl>
        <Input {...field} />
      </FormControl>
      <FormMessage />
    </FormItem>
  )}
/>
<FormField
  control={form.control}
  name={`items.${index}.answer`}
  render={({ field }) => (
    <FormItem>
      <FormLabel>Resposta</FormLabel>
      <FormControl>
        <Textarea rows={4} placeholder="Texto ou HTML simples (links com <a>, <strong>...)" {...field} />
      </FormControl>
      <FormMessage />
    </FormItem>
  )}
/>
```

Header:

```tsx
<DialogTitle>Editar FAQ</DialogTitle>
<DialogDescription>Configure perguntas e respostas.</DialogDescription>
```

Mensagem de erro do array root:

```tsx
{form.formState.errors.items?.message ? (
  <p className="text-sm text-destructive">{form.formState.errors.items.message}</p>
) : null}
```
