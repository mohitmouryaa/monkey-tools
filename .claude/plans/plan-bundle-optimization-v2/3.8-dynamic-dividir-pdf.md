# ✅ 3.8 — Tornar dynamic `pdf-lib` e `jszip` em `dividir-pdf.tsx`

## Contexto

Tool usa `pdf-lib` (linha 4: `import { degrees, PDFDocument } from "pdf-lib"`)
e adicionalmente `jszip` quando empacota múltiplos PDFs no resultado. Ambos
são usados no handler `splitPDF` (em torno da linha 64). Verificar se `jszip`
está mesmo importado top-level — pode estar faltando ou pode estar inline.

## Detalhe técnico

Em `apps/web/modules/tools/ui/components/dividir-pdf.tsx`:

1. **Listar imports atuais de `jszip` e `pdf-lib`**:
   - Linha 4: `import { degrees, PDFDocument } from "pdf-lib";` (confirmado)
   - Verificar se há `import JSZip from "jszip"` (grep)

2. **Remover imports top-level** dessas libs.

3. **Adicionar imports do util**:
   ```ts
   import { lazyLoadJsZip, lazyLoadPdfLib } from "@/modules/common/lib/lazy-load-libs";
   ```
   (Se `jszip` não está importado top-level, mas é usado via require/dynamic, adaptar conforme — mas o plano lista `jszip` como import a tornar dynamic, então deve haver uso.)

4. **Dentro de `splitPDF`** (handler em torno da linha 64), antes de `PDFDocument.create()` (linha 75), adicionar:
   ```ts
   const { PDFDocument, degrees } = await lazyLoadPdfLib();
   ```
   E onde `JSZip` é instanciado (procurar `new JSZip(`):
   ```ts
   const { default: JSZip } = await lazyLoadJsZip();
   const zip = new JSZip();
   ```
   Se `degrees` não é usado, omitir do destructuring.

## Arquivos tocados

- `apps/web/modules/tools/ui/components/dividir-pdf.tsx` — remove imports top-level de pdf-lib e jszip, dynamic load no handler
