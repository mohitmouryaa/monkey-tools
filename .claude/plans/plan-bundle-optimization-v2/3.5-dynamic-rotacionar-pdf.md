# ✅ 3.5 — Tornar dynamic `pdf-lib` em `rotacionar-pdf.tsx`

## Contexto

Tool usa `pdf-lib` apenas no handler de salvar (`savePDF`). Caso simples — só
um handler. Linha 4: `import { degrees, PDFDocument } from "pdf-lib";`. Sem
estado tipado com `PDFDocument`.

## Detalhe técnico

Em `apps/web/modules/tools/ui/components/rotacionar-pdf.tsx`:

1. **Remover linha 4**:
   ```ts
   import { degrees, PDFDocument } from "pdf-lib";
   ```

2. **Adicionar import do util**:
   ```ts
   import { lazyLoadPdfLib } from "@/modules/common/lib/lazy-load-libs";
   ```

3. **Dentro de `savePDF`** (handler em torno da linha 78), antes do primeiro `PDFDocument.create()` (linha 89), adicionar:
   ```ts
   const { PDFDocument, degrees } = await lazyLoadPdfLib();
   ```

4. Demais usos (`PDFDocument.create()`, `PDFDocument.load(...)`, `degrees(rotation)`) continuam funcionando — variáveis locais com mesmos nomes.

## Arquivos tocados

- `apps/web/modules/tools/ui/components/rotacionar-pdf.tsx` — remove import top-level de pdf-lib, dynamic load no handler

## Verificação

- [ ] `grep -n "from \"pdf-lib\"" apps/web/modules/tools/ui/components/rotacionar-pdf.tsx` retorna zero
- [ ] `grep -n "lazyLoadPdfLib" apps/web/modules/tools/ui/components/rotacionar-pdf.tsx` retorna 2 linhas (import + uso no handler)

## Decisões aplicadas

- **P3** (A — runtime carregado uma vez no handler): destructuring em uma linha.
