# ✅ 3.4 — Tornar dynamic `pdf-lib` em `numerar-pdf.tsx` (caso complexo: estado + helpers + preview + types)

## Contexto

Tool é o segundo caso complexo. Já tem `type PDFPage` e `type PDFFont` como
type-only na linha 4 (parcial). Falta tornar `PDFDocument`, `StandardFonts` e
`rgb` dynamic. Sites de uso:
1. `useState<PDFDocument | null>` (linha 48) — só tipo
2. `fontMap` global (linha 37) usa `StandardFonts.*` — runtime
3. `drawPageNumber` callback (linha 87) — usa `rgb(...)` runtime
4. `handleFileSelect` (linha 211) — `PDFDocument.load`
5. `handleDownload` (linha 245) — `PDFDocument.load(await pdfDoc.save())`, `embedFont`

## Detalhe técnico

Em `apps/web/modules/tools/ui/components/numerar-pdf.tsx`:

1. **Trocar linha 4** de:
   ```ts
   import { PDFDocument, StandardFonts, rgb, type PDFPage, type PDFFont } from "pdf-lib";
   ```
   para:
   ```ts
   import type { PDFDocument, PDFPage, PDFFont } from "pdf-lib";
   import { lazyLoadPdfLib } from "@/modules/common/lib/lazy-load-libs";
   ```

2. **`fontMap` global (linhas 37-44)** — não pode ficar como const top-level porque referencia `StandardFonts.*`. Mover para **dentro das funções** que usam (`drawPageNumber` e `handleDownload`), OU mudar para `Record<string, "Helvetica" | "Helvetica-Bold" | ...>` com strings literais (StandardFonts é apenas um enum de strings — pode ser hardcoded).
   - **Default escolhido**: substituir por strings literais, já que `StandardFonts.HelveticaBold === "Helvetica-Bold"` etc. (verificar runtime de pdf-lib confirma esses valores como strings literais).
   - Resultado:
     ```ts
     const fontMap: Record<string, string> = {
       Helvetica: "Helvetica",
       "Helvetica-Bold": "Helvetica-Bold",
       "Times-Roman": "Times-Roman",
       "Times-Bold": "Times-Bold",
       Courier: "Courier",
       "Courier-Bold": "Courier-Bold",
     };
     ```
   - Se preferir não hardcodar (fragilidade contra mudança de constantes em pdf-lib), alternativa: deixar `fontMap` retornar a chave que vai ser passada para `embedFont`, e fazer o lookup `pdfLib.StandardFonts[fontMap[...]]` dentro do handler. Escolher a forma mais simples; o test de smoke (3.10) confirma comportamento.

3. **`drawPageNumber` (linha 87)** — passa a receber `rgb` (ou `pdfLib`) como argumento:
   - Mudar assinatura para `(page: PDFPage, font: PDFFont, actualPageNumber: number, currentSettings: PageNumberSettings, rgb: typeof import("pdf-lib").rgb) => { ... }`
   - Substituir `rgb(0, 0, 0)` por `rgb(0, 0, 0)` (mesmo nome — argumento agora).

4. **`handleFileSelect` (linha 211)** — carregar pdf-lib antes:
   ```ts
   const arrayBuffer = await selectedFile.arrayBuffer();
   const { PDFDocument } = await lazyLoadPdfLib();
   const pdf = await PDFDocument.load(arrayBuffer);
   ```

5. **`handleDownload` (linha 245)** — carregar pdf-lib antes do uso:
   ```ts
   const pdfLib = await lazyLoadPdfLib();
   const pdfDocCopy = await pdfLib.PDFDocument.load(await pdfDoc.save());
   const fontName = fontMap[settings.fontFamily] ?? "Helvetica-Bold";
   const font = await pdfDocCopy.embedFont(fontName);

   for (let i = settings.pageRange.from - 1; i < settings.pageRange.to; i++) {
     if (i >= 0 && i < pdfDocCopy.getPageCount()) {
       drawPageNumber(pdfDocCopy.getPage(i), font, i + 1, settings, pdfLib.rgb);
     }
   }
   ```

6. **`renderPreview`** (busca por `tempPdf` em torno da linha 148) — mesmo tratamento: carregar `pdfLib` no início, usar `pdfLib.PDFDocument.create()`, passar `pdfLib.rgb` para `drawPageNumber`.
