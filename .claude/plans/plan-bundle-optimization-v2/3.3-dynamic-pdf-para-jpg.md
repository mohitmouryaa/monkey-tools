# ✅ 3.3 — Tornar dynamic `jszip` em `pdf-para-jpg.tsx`

## Contexto

`pdfjs-dist` JÁ é dynamic (linha 82 do arquivo: `const pdfjsLib = await import("pdfjs-dist")`). O único import eager é `JSZip` (linha 3, ~153 KB).

## Detalhe técnico

Em `apps/web/modules/tools/ui/components/pdf-para-jpg.tsx`:

1. **Remover linha 3**:
   ```ts
   import JSZip from "jszip";
   ```

2. **Adicionar import do util** (próximo aos outros imports `from "@/modules/common/..."`):
   ```ts
   import { lazyLoadJsZip } from "@/modules/common/lib/lazy-load-libs";
   ```

3. **Localizar o uso de `new JSZip()`** (em torno da linha 182 conforme grep prévio). Antes da primeira referência, adicionar:
   ```ts
   const { default: JSZip } = await lazyLoadJsZip();
   const zip = new JSZip();
   ```
   Substituindo a linha existente que faz `const zip = new JSZip();`.

4. Os usos seguintes (`zip.file(...)`, `zip.generateAsync(...)`) continuam funcionando porque `zip` é a mesma variável local.
