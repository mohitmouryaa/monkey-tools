# ✅ 3.0 — Criar util `lazy-load-libs.ts` para carregar pdf-lib, xlsx, jspdf e jszip sob demanda

## Contexto

Hoje as tools fazem `await import("pdf-lib")` espalhado em handlers, com risco
de duplicar o `Promise` retornado a cada clique. Centralizar num util dá:
(a) cache do módulo via Promise singleton, (b) ponto único de manutenção, (c)
mesma assinatura para as 4 libs.

## Detalhe técnico

Criar arquivo `apps/web/modules/common/lib/lazy-load-libs.ts` exportando 4
funções, cada uma cacheando o Promise retornado em variável module-level.

```ts
let pdfLibPromise: Promise<typeof import("pdf-lib")> | null = null;
let xlsxPromise: Promise<typeof import("xlsx")> | null = null;
let jsPdfPromise: Promise<typeof import("jspdf")> | null = null;
let jsZipPromise: Promise<typeof import("jszip")> | null = null;

export function lazyLoadPdfLib() {
  pdfLibPromise ??= import("pdf-lib");
  return pdfLibPromise;
}

export function lazyLoadXlsx() {
  xlsxPromise ??= import("xlsx");
  return xlsxPromise;
}

export function lazyLoadJsPdf() {
  jsPdfPromise ??= import("jspdf");
  return jsPdfPromise;
}

export function lazyLoadJsZip() {
  jsZipPromise ??= import("jszip");
  return jsZipPromise;
}
```

Notas de uso (incluir como JSDoc breve em cada export):
- Para `lazyLoadPdfLib`: `const { PDFDocument, rgb, degrees, StandardFonts } = await lazyLoadPdfLib();`
- Para `lazyLoadJsPdf`: `const { default: jsPDF } = await lazyLoadJsPdf();` (jspdf usa default export em ESM)
- Para `lazyLoadXlsx`: `const XLSX = await lazyLoadXlsx();` (módulo namespace)
- Para `lazyLoadJsZip`: `const { default: JSZip } = await lazyLoadJsZip();` (jszip usa default export)
