# ✅ 3.1 — Tornar dynamic `xlsx`, `jspdf` e `pdf-lib` em `excel-para-pdf.tsx`

## Contexto

Tool já tem `pdf-lib` parcialmente dynamic (linha 57) mas `xlsx` (~399 KB) e
`jspdf` (~407 KB) ainda são imports estáticos no topo. Refatorar para usar o
util criado em 3.0 e garantir que NENHUMA dessas libs entre no bundle inicial
da rota de tool.

## Detalhe técnico

Em `apps/web/modules/tools/ui/components/excel-para-pdf.tsx`:

1. **Remover** as linhas 4 e 5:
   ```ts
   import * as XLSX from "xlsx";
   import { jsPDF } from "jspdf";
   ```

2. **Adicionar** import do util:
   ```ts
   import { lazyLoadJsPdf, lazyLoadPdfLib, lazyLoadXlsx } from "@/modules/common/lib/lazy-load-libs";
   ```

3. **Dentro de `convertToPdf`** (handler que começa por volta da linha 45), substituir:
   - O `await import("pdf-lib")` da linha 57 por `await lazyLoadPdfLib()`
   - Adicionar no início do `try` (antes do `for` da linha 61):
     ```ts
     const XLSX = await lazyLoadXlsx();
     const { default: jsPDF } = await lazyLoadJsPdf();
     const { PDFDocument } = await lazyLoadPdfLib();
     ```
   - Remover a linha 57 antiga (`const { PDFDocument } = await import("pdf-lib");`)

4. Os usos `XLSX.read(...)`, `XLSX.utils.sheet_to_json(...)`, `new jsPDF("landscape")`, `PDFDocument.create()`, `PDFDocument.load(...)` continuam funcionando porque as variáveis locais têm os mesmos nomes.
