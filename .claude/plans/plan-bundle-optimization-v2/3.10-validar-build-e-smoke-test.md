# ✅ 3.10 — Validar build com analyzer e smoke test em todas as 9 tools

## Contexto

Critério de conclusão da Fase 3 exige que `/ferramentas/[toolCategory]/[tool]`
caia abaixo de 1300 KB e que cada tool ainda funcione. Esta task fecha o ciclo:
roda o build com analyzer, valida tamanho, valida funcionamento manual.

## Detalhe técnico

### 1. Typecheck e lint

Rodar:
```bash
pnpm --filter web typecheck
pnpm lint
```

Ambos devem passar zero erros novos.

### 2. Build com analyzer

```bash
ANALYZE=true pnpm --filter web build
```

(Se o wrapper de bundle-analyzer não estiver no `next.config.mjs`, pode ser
necessário adicionar — mas Fase 1.2 já fez isso. Confirmar via leitura do
arquivo.)

Coletar do output:
- Tamanho de `/ferramentas/[toolCategory]/[tool]` (First Load JS)
- Verificar se chunks de `pdf-lib`, `xlsx`, `jspdf`, `jszip` aparecem como chunks separados (não dentro do bundle inicial da rota)
- Comparar com baseline antes da Fase 3 (linha do plano: 2564 KB → meta < 1300 KB)

### 3. Smoke test funcional (dev server)

Subir dev: `pnpm dev` (se não estiver rodando).

Para CADA tool da fase, fazer um caminho feliz mínimo:

- [ ] `/ferramentas/.../excel-para-pdf` — upload .xlsx pequeno → "Converter para PDF" → baixa
- [ ] `/ferramentas/.../marca-dagua-pdf` — upload PDF pequeno → preview aparece com marca → muda configuração → preview atualiza → baixa
- [ ] `/ferramentas/.../pdf-para-jpg` — upload PDF de 2 páginas → converter → ZIP com 2 JPGs
- [ ] `/ferramentas/.../numerar-pdf` — upload PDF → preview com número → ajustar posição → baixar
- [ ] `/ferramentas/.../rotacionar-pdf` — upload PDF → selecionar páginas → rotacionar → baixar
- [ ] `/ferramentas/.../mesclar-pdf` — upload 2 PDFs → mesclar → baixar
- [ ] `/ferramentas/.../json-para-excel` — colar JSON → exportar → arquivo .xlsx baixa
- [ ] `/ferramentas/.../dividir-pdf` — upload PDF → selecionar páginas → dividir → baixar
- [ ] `/ferramentas/.../jpg-para-pdf` — upload 2 JPGs → converter → PDF baixa

Em caso de erro em qualquer tool: PARAR e reportar a falha (a fase inteira não
fecha).

### 4. Documentar no PR

Escrever na descrição do PR (ou comentar no plano caso ainda não exista PR):

