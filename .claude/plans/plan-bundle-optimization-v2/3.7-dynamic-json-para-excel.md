# ✅ 3.7 — Tornar dynamic `xlsx` em `json-para-excel.tsx`

## Contexto

Tool importa `xlsx` (~399 KB) no topo (linha 3) mas só usa em um único handler
(`exportToExcel` em torno da linha 145, com `XLSX.utils.book_new()`,
`XLSX.utils.json_to_sheet`, `XLSX.utils.book_append_sheet`, `XLSX.writeFile`).

## Detalhe técnico

Em `apps/web/modules/tools/ui/components/json-para-excel.tsx`:

1. **Remover linha 3**:
   ```ts
   import * as XLSX from "xlsx";
   ```

2. **Adicionar import do util**:
   ```ts
   import { lazyLoadXlsx } from "@/modules/common/lib/lazy-load-libs";
   ```

3. **No handler que chama `XLSX.utils.book_new()`** (provavelmente `exportToExcel` ou similar), antes do primeiro uso de `XLSX.*`, adicionar:
   ```ts
   const XLSX = await lazyLoadXlsx();
   ```
   Garantir que o handler é `async` (provavelmente já é por causa do `await`; verificar e ajustar se necessário).

4. Demais usos (`XLSX.utils.book_new()`, `XLSX.writeFile(...)`, etc.) continuam funcionando — variável local com mesmo nome.

## Arquivos tocados

- `apps/web/modules/tools/ui/components/json-para-excel.tsx` — remove import top-level de xlsx, dynamic load no handler
