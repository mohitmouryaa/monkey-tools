# ✅ 3.2 — Tornar dynamic `pdf-lib` em `marca-dagua-pdf.tsx` (caso complexo: estado + helpers + preview)

## Contexto

Tool é o caso mais complexo: importa `PDFDocument`, `rgb`, `StandardFonts`,
`degrees` no topo e usa em 4 sites distintos:
1. `useState<PDFDocument | null>(null)` (linha 33) — só tipo
2. `handleFileSelect` (linha 78) — `PDFDocument.load`
3. `addWatermarkToPage` callback (linha 108) — `StandardFonts`, `rgb`, `degrees`, `embedFont`
4. `renderPreview` (linha 220) — `PDFDocument.create`

Se simplesmente trocar para `await import` em cada site, perdemos tipagem.
Solução: `import type` no topo (zero runtime) + `await lazyLoadPdfLib()` em
cada handler que precisa do runtime, passando os símbolos como argumentos para
helpers.

## Detalhe técnico

Em `apps/web/modules/tools/ui/components/marca-dagua-pdf.tsx`:

1. **Trocar linha 10** de:
   ```ts
   import { PDFDocument, rgb, StandardFonts, degrees } from "pdf-lib";
   ```
   para:
   ```ts
   import type { PDFDocument } from "pdf-lib";
   import { lazyLoadPdfLib } from "@/modules/common/lib/lazy-load-libs";
   ```

2. **`addWatermarkToPage` (linha 108)** — passa a receber `pdfLib` como argumento:
   - Mudar assinatura de `async (pdf: PDFDocument, pageIndex: number) => { ... }` para `async (pdf: PDFDocument, pageIndex: number, pdfLib: typeof import("pdf-lib")) => { ... }`
   - Dentro, substituir referências:
     - `StandardFonts.Helvetica` → `pdfLib.StandardFonts.Helvetica` (e demais)
     - `rgb(...)` → `pdfLib.rgb(...)`
     - `degrees(...)` → `pdfLib.degrees(...)`
   - Atualizar dependência do `useCallback` ou converter em função regular se a deps array ficar awkward. **Default**: manter `useCallback` e adicionar nada nas deps (pdfLib é argumento, não captura).

3. **`handleFileSelect` (linha 78)** — carregar pdf-lib antes de usar:
   ```ts
   const arrayBuffer = await file.arrayBuffer();
   const { PDFDocument } = await lazyLoadPdfLib();
   const pdf = await PDFDocument.load(arrayBuffer);
   ```

4. **`renderPreview` (linha 220)** — idem:
   ```ts
   const pdfLib = await lazyLoadPdfLib();
   const tempPdf = await pdfLib.PDFDocument.create();
   const [copiedPage] = await tempPdf.copyPages(pdfDoc, [currentPage - 1]);
   tempPdf.addPage(copiedPage);
   if (currentPage >= pageRange.from && currentPage <= pageRange.to) {
     await addWatermarkToPage(tempPdf, 0, pdfLib);
   }
   ```

5. **Submit handler `processWatermark`** (em torno da linha 261, busca pelo `PDFDocument.create()` da watermarkedPdf) — mesmo tratamento: `const pdfLib = await lazyLoadPdfLib(); const watermarkedPdf = await pdfLib.PDFDocument.create();` e passar `pdfLib` para `addWatermarkToPage`.

6. Onde quer que o código use `PDFDocument` como **tipo** (state, parâmetro de função), continua funcionando porque o `import type` foi mantido.
